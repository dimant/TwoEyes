# Two Eyes — Project Plan

A go (baduk) puzzle app for **complete beginners**, built entirely from **public-domain /
generated content**, that works **offline** (installable PWA) and **validates** solving
attempts. Name: *two eyes* = a group with two eyes lives.

- **Repo:** `git@github.com:dimant/TwoEyes.git` (`main` tracks `origin/main`)
- **Stack:** React + Vite + vite-plugin-pwa · TypeScript (ESM) · MVVM, no state library
  (`Observable` + `useSyncExternalStore`) · Vitest + jsdom · content generated by a
  standalone seeded CLI + go rules engine (never shipped to the client)
- **Tests:** 706 passing · `npm run typecheck`, `npm run build`, offline all verified

---

## What we have today

### Content pipeline (build-time only)
- **Go rules engine** — `src/engine/` (Board, liberties flood-fill, play/capture/suicide).
- **Bounded capture-reader** — `src/generator/reader.ts` (`capturedUnderBestPlay`) for
  verifying nets/ladders; `snapbackWorks` replay for snapbacks.
- **Construct-and-verify generators** — `src/generator/` with seeded RNG, dedup, curation,
  fail-loud guards. `npm run generate` regenerates deterministically.
- **Committed bank** — `src/bank/bank.json`, engine-verified by a permanent solvability
  suite. **440 puzzles**: 11 topics × 2 rungs × 20.

### App
- **MVVM core** — `src/app/model/` (types, `PuzzleBank`, `checkAnswer`, `ProgressStore`) and
  `src/app/vm/` (`PlayerViewModel` state machine, `MapViewModel`). No React/engine imports.
- **UI** — `Board.tsx` (SVG goban), `PlayerScreen`, `MapScreen`, `LessonScreen`, inputs,
  feedback. Approved visual direction (goban tan, slate/shell stones, teal accent), light/dark tokens.
- **Progression** — mastery = 4 clears a rung; linear-with-preview unlock; **triple-tap
  skip-ahead** (touch-reliable, persistently unlocks the tapped topic + all prior ones).
- **Concept lessons** — every topic opens with a short **engine-verified illustrated
  lesson**; auto-shows on first entry, reopenable via a **Learn** button.
- **PWA** — installable, offline precache of app shell + bundled bank.

## Roadmap

Phases are ordered by value. Each is a self-contained chunk producing working, testable
software. Detailed specs/plans live under `docs/superpowers/`.

### Phase 5 — Stage B.2: sequence engine + ladders  ✅ *shipped*

> **5a shipped** — nets/snapbacks reveal their capture move-by-move. **5b.1 shipped** — ladder (8) generator + animated payoff. **5b.2 shipped** — ladder-breaker (9), Q-binary recognition + breaker highlight. **Stage B complete** (topics 1–11 all ship). The interactive "you play → engine responds" player mode was consciously dropped in the 5b.1 pivot (ladders branch, so pick-the-start beat hand-play-every-move).

The spine of the roadmap: turn the app from "solve one move" into "read a sequence."
- **Sequence engine** — solution *trees* + an interactive "you play → engine responds →
  repeat (with refutation)" player mode.
- **Ladder (8)** and **Ladder-breaker (9)** generators (reuse the capture-reader).
- **Animated payoff** — Net/Snapback lessons and puzzle reveals play the capture out
  move-by-move instead of a static hint.
- *Largest build; on the scale of Stage B.1.*

### Phase 6 — Interaction & UX polish  🔜 in progress
- **Animate the capture on reveal** ✅ *shipped* — capture topics (2, 3, and the
  capturing puzzles of 7) reveal through the stepped `PayoffBoard` (Next move ▸ / Replay)
  instead of snapping; the 1-move payoff is derived at the view (no bank change).
- **"Lessons" browser on the map** ✅ *shipped* — a per-card **Learn** button opens
  any concept lesson (locked topics included) as a full-screen takeover over the map;
  viewing marks it seen so first practice entry skips the auto-lesson.
- **Light/dark theme toggle** ✅ *shipped* — a cycling top-bar button (System → Light →
  Dark), persisted; `data-theme` on the root drives the already-theme-aware CSS, with a
  no-FOUC inline script.
- Keyboard-accessible board input (separate spec, not yet started).

### Phase 7 — Content enrichment  📋 planned
- **Topic 4 (Escape atari)** — enrich the weakest, structurally monotonous generator
  (e.g. 2-stone groups, varied escape shapes).
- **Smith 1908 tier** — Arthur Smith's *The Game of Go* (public domain, ~99 audited
  problems) as a curated enrichment set beyond the generated bank.

### Phase 8 — Stages C–D (topics 12–21)  💡 backlog
- Life & death, eye shape, endgame, etc. Most topics need a **life-and-death solver**
  (bigger than the current reader). Gated on Phase 5's tree engine as a foundation.

## Principles (kept throughout)
- **Public-domain / generated by construction** — nothing shipped that isn't ours or PD.
- **Everything verified** — all puzzle and lesson content is proven by the engine in a
  permanent test suite; no shape ships on trust.
- **Disciplined workflow** — brainstorm → spec → plan → build (tests + review) per chunk.
- **Beginner-first** — small boards, one idea at a time, teach before drilling.
